# Grids

Three types of grids are at your disposal: Default grids, Asymmetric grids and Custom grids.

`.grid` is frequently a child of a `.container` or a `.container__[child]` element.

# Responsive Grids
Responsive grids propose standard alignments and default responsiveness with a minimal amount of CSS (classes are defined only on the parent). 

Ideal to align content in a `container`, for example a list of Card elements. 

Use the following classes:
- `grid grid--responsive-cols-2`
- `grid grid--responsive-cols-3`
- `grid grid--responsive-cols-4`

Additionally, a `gap--responsive` class applies default spacings between elements.

# Asymmetric Grids
Asymmetric grids are responsive grids, but the columns have differentiated widths. This can be used to display images and content on two cols.
- `grid grid--responsive-cols-1/4-3/4`
- `grid grid--responsive-cols-3/4-1/4`
- `grid grid--responsive-cols-1/3-2/3`
- `grid grid--responsive-cols-2/3-1/3`

An additional `grid grid--reverse` class helps you to flip right and left columns

# Custom Grids

You can use most of [Tailwind grid classes](https://tailwindcss.com/docs/grid-template-columns) to display your elements in a more specific way. You will have create a template in this project with the new classes or safelist the desired Tailwind classes, and to build the css file again.
